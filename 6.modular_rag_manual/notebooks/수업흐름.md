# LangGraph 기반 Modular RAG 모듈

# 데이터 소개
```text
이 문서는 규모가 작기 때문에 실제로는 Naive RAG만으로도 어느 정도 동작합니다. 그러나 문서 안에 반복전화, 폭언, 폭행, 위험물 소지, 보호조치 등 상황 유형이 명확하게 나뉘어 있기 때문에 Modular RAG의 구조를 설명하기 좋은 실습 데이터입니다. 즉, 이 데이터는 Modular RAG가 필수인 데이터라기보다, Modular RAG 설계를 학습하기에 적합한 데이터입니다.
```

# 13번 예제
## LangGraph 노드별 역할
| 노드                  | 역할                                                                             | 주요 입력                                         | 주요 출력                                 | 다음 흐름                                                 |
| ------------------- | ------------------------------------------------------------------------------ | --------------------------------------------- | ------------------------------------- | ----------------------------------------------------- |
| `start`             | 그래프 실행 시작점이다. 사용자의 질문이 들어오면 전체 RAG 파이프라인을 시작한다.                                | 사용자 질문                                        | 초기 State                              | `query_rewrite`                                       |
| `query_rewrite`     | 사용자 질문을 검색에 적합한 형태로 재작성한다. 모호한 질문을 매뉴얼 검색에 유리한 키워드 중심 질문으로 바꾼다.                | `question`, `rewrite_count`                   | `rewritten_question`, `rewrite_count` | `retrieve_manual`                                     |
| `retrieve_manual`   | 재작성된 질문을 기준으로 `공직자_민원응대_핵심_매뉴얼`에서 관련 문서를 검색한다. Vector DB에서 Top-k 문서를 가져오는 단계다. | `rewritten_question`                          | `documents`                           | `grade_documents`                                     |
| `grade_documents`   | 검색된 문서가 질문에 답하기에 충분한지 평가한다. 관련성이 낮으면 질문 재작성 단계로 되돌린다.                          | `question`, `rewritten_question`, `documents` | `document_grade`, `route_reason`      | 충분하면 `organize_evidence`, 부족하면 `query_rewrite`        |
| `organize_evidence` | 검색된 문서 중 답변에 사용할 핵심 근거를 정리한다. 매뉴얼의 조치 절차, 대응 원칙, 유의사항 등을 답변 생성에 적합한 형태로 구조화한다. | `documents`                                   | `evidence`                            | `generate_answer`                                     |
| `generate_answer`   | 정리된 근거를 바탕으로 사용자 질문에 대한 답변을 생성한다. 매뉴얼 기반 답변이므로 임의 추론보다 근거 기반 설명을 우선한다.         | `question`, `evidence`                        | `answer`                              | `evaluate_answer`                                     |
| `evaluate_answer`   | 생성된 답변이 질문에 충분히 답했는지 평가한다. 답변이 부족하면 재검색 또는 질문 재작성 단계로 되돌린다.                    | `question`, `answer`, `evidence`              | `answer_grade`, `route_reason`        | 충분하면 `end`, 부족하면 `retrieve_manual` 또는 `query_rewrite` |
| `end`               | 답변 생성과 검증이 완료된 상태다. 최종 답변을 사용자에게 반환한다.                                         | `answer`                                      | 최종 답변                                 | `__end__`                                             |
| `__end__`           | LangGraph 내부 종료 노드다. 그래프 실행이 완전히 종료된다.                                         | 최종 State                                      | 실행 종료                                 | 없음                                                    |


## 흐름 기준으로 정리

| 단계 | 노드                  | 핵심 의미           |
| -: | ------------------- | --------------- |
|  1 | `query_rewrite`     | 질문을 검색 친화적으로 바꿈 |
|  2 | `retrieve_manual`   | 매뉴얼에서 관련 문서 검색  |
|  3 | `grade_documents`   | 검색 문서가 충분한지 판단  |
|  4 | `organize_evidence` | 답변에 사용할 근거 정리   |
|  5 | `generate_answer`   | 근거 기반 답변 생성     |
|  6 | `evaluate_answer`   | 답변 품질 평가        |
|  7 | `end`               | 충분한 답변이면 종료     |

## 조건 분기 구조
| 조건 판단 위치          | 조건         | 이동 방향               | 의미                  |
| ----------------- | ---------- | ------------------- | ------------------- |
| `grade_documents` | 검색 문서가 충분함 | `organize_evidence` | 검색 결과를 근거로 답변 생성 가능 |
| `grade_documents` | 검색 문서가 부족함 | `query_rewrite`     | 질문을 다시 고쳐 재검색 필요    |
| `evaluate_answer` | 답변이 충분함    | `end`               | 최종 답변으로 확정          |
| `evaluate_answer` | 답변이 부족함    | `retrieve_manual`   | 같은 질문으로 문서를 다시 검색   |
| `evaluate_answer` | 답변이 계속 부족함 | `query_rewrite`     | 질문 자체를 다시 재작성한 뒤 검색 |
